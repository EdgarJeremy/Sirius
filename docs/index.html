Introduction

Framework backend berbasis Model-Controller yang berfokus pada pengembangan aplikasi RESTful API. Sirius merepresentasikan tiap response dalam tipe konten http application/json dan transportasi response HTTP code seperti biasa layaknya response HTTP pada umumnya.

Sirius melakukan transport url melalui routing seperti sifat framework Codeigniter, hanya saja istilah untuk controller diganti dengan basepoint dan method class controller diganti dengan istilah endpoint

Perbandingan routing CI dengan Sirius

Codeigniter :
http://{domain}/index.php/{controller}/{method}/{param1}/{param2}/.../{paramN}

Sirius :
http://{domain}/index.php/{basepoint}/{endpoint}/{param1}/{param2}/.../{paramN}

Standard struktur response

Sirius memiliki standar response berupa objek JSON yang memiliki nilai masing-masing. Struktur-struktur ini adalah standar yang nantinya perlu diketahui seorang front-end atau pengembang aplikasi multiplatform agar dapat dilakukan penyesuaian dalam penggunaan response

  1. Response Operasi Sukses
    2.  Operasi sukses ditandai dengan response HTTP status code 200 dan property status adalah true, berikut contoh struktur JSONnya 
    {
						"status": true,
						"data": {
							"nama_aplikasi": "Sirius Framework",
							"versi": "0.0.2",
							"pesan": "Selamat bekerja!"
						},
						"f_method": "GET",
						"f_remote_ip": "::1",
						"waktu_eksekusi": 0.000100000000000000004792173602385929598312941379845142364501953125
					}
    Response operasi sukses juga akan diikuti dengan property data yang akan berisi data response dari backend sesuai dengan kebutuhan endpoint
  2. Response Operasi Gagal
    3.  Operasi gagal ditandai dengan response HTTP status code 200 dan property status bernilai false, berikut contoh struktur JSONnya 
    					{"status":false,"message":"Operasi yang anda inginkan gagal dijalankan","f_method":"GET","f_remote_ip":"::1","waktu_eksekusi":0.000100000000000000004792173602385929598312941379845142364501953125}
    Operasi gagal juga ditandai dengan property message yang berisi pesan kegagalan yang berkaitan dengan operasi tersebut, pesan tersebut ditentukan dari kode backend yang bersangkutan.
  3. Response Server/PHP Error
    4.  Response Server/PHP error ditandai dengan response HTTP status code 500 dan property status bernilai string "PHPERROR", berikut contoh struktur JSONnya 
    						{
							"status": "PHPERROR",
							"errors": [
								{
									"type": "E_PARSE",
									"message": "syntax error, unexpected '->' (T_OBJECT_OPERATOR)",
									"file": "/Applications/XAMPP/xamppfiles/htdocs/sirius_dev/sirius-0.0.2/app/basepoints/api.php",
									"line": 47,
									"code": 4
								}
							]
						}
    Response Server/PHP error juga disertai dengan property errors yang berisikan array dari error-error yang terjadi saat runtime
